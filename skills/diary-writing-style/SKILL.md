# Diary Writing Style

Guide users to write diary entries that feel real, readable, and reflective. Preserve the user's lived details and voice, then improve structure, rhythm, and insight.

## Core Principles

1. Meaning before chronology: organize by significance, not by clock time.
2. Detail before abstraction: keep concrete moments, reduce generic statements.
3. Reflection before conclusion: add one layer of personal interpretation.
4. Action before slogan: end with a concrete next step.

## Workflow

Use this five-step loop for every entry.

1. Capture source material
   - Ask for raw notes or draft text.
   - If no text is provided, ask for 3-5 moments from the day.

2. Find the anchor
   - Select one central thread (trip, conversation, setback, insight).
   - Keep only events that support this thread.

3. Build paragraph intent
   - Paragraph 1: opening moment and scene.
   - Paragraph 2-4: key events with concrete detail.
   - Paragraph 5: what changed in the writer's perspective.
   - Last paragraph: one practical action for tomorrow.

4. Rewrite with voice preservation
   - Keep first-person voice and personality markers.
   - Smooth run-on sentences and remove repetitive connectors.
   - Convert "event list" into "event + feeling + interpretation".

5. Run anti-"flowing account" pass
   - Remove trivial timeline transitions.
   - Keep one memory point per paragraph.
   - Replace vague words with specific objects, prices, places, or dialogue.

## Rewrite Modes

Pick one mode based on user intent.

- Light polish: minimal edits, preserve wording as much as possible.
- Structural rewrite: re-order paragraphs for stronger narrative flow.
- Coaching mode: do not fully rewrite; explain changes and give drills.

## Style Rules

- Write in active voice.
- Prefer short-to-medium sentence length.
- Use contrast patterns: "I thought... but...", "Originally... later...".
- Keep emotional tone restrained; avoid over-dramatic conclusions.
- Avoid moralizing language; prioritize observation.
- End with a specific commitment, not a motivational slogan.

## Output Contract

When asked to optimize an entry, return:

1. Revised text (full version)
2. Why it reads better (3-5 bullets)
3. Reusable writing pattern for next entry (1 short template)

For reusable templates and drills, read:
- `references/templates.md`
- `references/revision-checklist.md`

## Constraints

- Do not invent events, people, or facts.
- Do not flatten the user's voice into generic "perfect prose".
- Do not over-compress if the user wants diary texture.
- Do not turn the entry into pure self-help content.

- "优化下这篇日记"

- "保留我的语气，重写得更顺"
