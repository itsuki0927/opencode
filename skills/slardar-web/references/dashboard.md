# Dashboard — 完整参考

看板管理命令的完整参数参考。涵盖列表查询、详情获取、数据查询、收藏管理。

## dashboard list — 看板列表

列出指定 bid 下的看板，支持关键词搜索和收藏筛选。

### 完整参数表

| 参数          | 必须 | 默认值   | 说明                   |
| ------------- | ---- | -------- | ---------------------- |
| `--bid`       | 是   | -        | 业务 ID                |
| `--liked`     | 否   | false    | 仅显示收藏的看板       |
| `--keyword`   | 否   | -        | 按关键词搜索看板名称   |
| `--page`      | 否   | 1        | 页码                   |
| `--page-size` | 否   | 999      | 每页条数（999 即全部） |
| `--env`       | 否   | 默认 env | 环境                   |
| `--raw`       | 否   | false    | 输出原始 JSON          |

## dashboard get — 看板详情

获取看板的配置结构，包括图表列表、指标配置、全局设置等元信息。

### 完整参数表

| 参数             | 必须 | 默认值     | 说明          |
| ---------------- | ---- | ---------- | ------------- |
| `--dashboard-id` | 是   | -          | 看板 ID       |
| `--bid`          | 是   | -          | 业务 ID       |
| `--env`          | 否   | production | 环境          |
| `--raw`          | 否   | false      | 输出原始 JSON |

### 输出内容说明

- **基本信息**：看板 ID、创建者、看板类型、收藏状态、时间范围、直链
- **全局设置**：看板级别的全局配置项
- **图表列表**：每个图表的名称及其包含的指标
- **类型统计**：各图表类型的数量汇总

## dashboard query — 看板数据查询

获取看板中所有图表的实际数据，包括指标卡当前值和环比、趋势图时序数据、数据表。

### 完整参数表

| 参数             | 必须 | 默认值   | 说明                                                     |
| ---------------- | ---- | -------- | -------------------------------------------------------- |
| `--dashboard-id` | 是   | -        | 看板 ID                                                  |
| `--bid`          | 是   | -        | 业务 ID                                                  |
| `--env`          | 否   | 自动检测 | 环境（不传时自动调 get_online 获取线上环境）             |
| `--all-env`      | 否   | false    | 查所有环境（不带 env 参数）                              |
| `--start-time`   | 否   | -        | 起始时间 Unix 时间戳（秒），需与 `--end-time` 一起使用   |
| `--end-time`     | 否   | -        | 结束时间 Unix 时间戳（秒），需与 `--start-time` 一起使用 |
| `--granularity`  | 否   | 自动     | 时间粒度（秒），如 60/300/1800/3600/86400                |
| `--raw`          | 否   | false    | 输出原始 JSON                                            |

### 环境参数说明

- 不传 `--env` → 自动使用该 bid 的线上环境（推荐）
- `--env <value>` → 使用指定环境
- `--all-env` 或 `--env Slardar_All` → 查所有环境

### 时间筛选说明

- 用户指定时间范围（如"最近7天"、"昨天"、"上周"等自然语言）时，**必须**先调用 `slardar-web-cli time parse` 将其转为 Unix 时间戳，再通过 `--start-time` + `--end-time` 传入。**禁止**自行编造时间戳数值。
- `--start-time` 和 `--end-time` 需成对使用。
- 不传时间参数时，使用看板自身配置的默认时间范围。如果看板使用自定义时间且起始时间距今超过 3 个月，会自动 fallback 到最近 3 天。
- `--granularity` 可与时间参数组合使用，也可独立使用（仅改粒度，不改时间范围）。不传时根据时间跨度自动选择。

### 输出说明

输出按图表类型分区展示：

- **指标卡片**：当前值 + 环比变化（可能有）
- **趋势图**：各系列汇总统计 + 时序数据表（系列数超过 20 个时自动截断，展示 Top 20）
- **数据表**：如 TOP JS 错误等，自动聚合

## 注意事项

1. **dashboard_type 说明**：`dashboard_type=0` 为常规看板，`dashboard_type=1` 为长期观测看板。
2. **详情 vs 数据区别**：`dashboard get` 返回看板的配置结构（有哪些图表、用了什么指标）；`dashboard query` 返回图表的实际数据值。
3. **只读限制**：本技能仅支持查询和查看操作，不支持创建、修改、删除看板或编辑图表配置。如用户需要写操作，应提示前往 Slardar 控制台操作。
4. **忽略链接中的时间参数**：看板链接中可能携带 `start_time` 和 `end_time` 参数，应忽略这些参数，不要将其作为查询的时间范围。
